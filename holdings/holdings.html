*OCLC API Known Issue:
When an OCLC number has been merged under another record, the merged OCLC number is in the 019 field of that bibliographic record. ... For Library Catalog Resource requests, the merged OCLC number only works correctly if frbrGrouping is set to "on" or on by default. When the frbrGrouping parameter is set to "off" the merged OCLC number request incorrectly results in indicating there are no holdings.
